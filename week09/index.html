Transform & Transition: More Fun

Transform Clock
Inspired by Wes Bos I wanted to combine transforms and transitions with sound after the team activity this week. This mobile friendly JS clock has some fun transition beziers. I added a tock sound file that plays with every transition, but only if the user interacts with the page (at least in Chromium browsers) due to Google's clampdown on unwanted audio. So, click if you want a little tick with your clock. In order to get the transitions not to freak out, I had to rework the way the angles were being computed. There should be no clock hand freakouts with this implementation. Check out : the clock CSS properties of note:
transform-origin: 100%; transition-timing-function: cubic-bezier(1, -1.39, 0, 3.26);
JS of note:
// audioElement.play() returns a promise. // This is the code that shuts off the audio message // volume control! sound.volume = .1; sound.play() .then((res) => { if(msgDiv.style.display != 'none') { // a success message is logged here, but you don't want it going off all the time, hence the style check msgDiv.style.display = 'none'; } }, (res)=>{ // on promise reject, turning on the audio message msgDiv.style.display = 'flex-box'; });

Element by Attribute

Query an element by attribute state
This deserves special emphasis given the utility of it. Select an element based on an attribute's value through the NodeQuery Interface! Using: document .querySelector( "CSSSelector[attribute='match']" ); works with querySelectorAll() too. For example: <audio data-key='keyP' src='/path/sum.wav'> Select with: .querySelector(`audio[data-key='keyP']`) With ES6 string templates go one step further:
// < - scroll - > document .querySelector( `audio[data-key="${e.property}"]` ); // this could be in an callback function // which takes an event. Pretty Cool! // NOTE that the array bracket goes around // the property, equality operator (=) // and template property here // AND template variable needs to be within quotes.
