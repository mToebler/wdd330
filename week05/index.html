Closures

Controlling Scope
Closers control what is and isn't in function scope. Last week, I used a closer in the WeakMap example of OOP encapsulation. With closures, inner function can gain access to the additional, supplied scope of the en-closing function. This allows for: - Encapsulation - Data privacy Classic example:
const getSecret = (secret) => { return { get: () => secret }; }; const key = getSecret('private'); key.get(); // returns private
The privleged method get() will grant access to secret set via with the returned object. Next, we'll explore partial application and currying with closures.

Use strict
Simple enough:
"use strict"; // first line in file
Use function scope:
function showStrict() { "use strict"; // first line in function body }
Recommended for self-invoking functions:
(function() { "use strict"; // 1st line in body ... }());

Prototypal Inheritance

Duner PROTO
All instances of a "class" share its dunder proto: object.__proto__; which is the same as: object.constructor.prototype;
__proto__ is live!
Changing one will change all. Example:(<= scroll =>)
simple = {}; simple_1 = new simple.__proto__.constructor(); // new isn't necessary? simple_2 = simple_1.__proto__.constructor(); simple_2.__proto__.property = '6'; simple.property === '6'; // true // these inherited properties can be overridden without affecting any of the others. simple.property = 1; // overriding will them claim it as its own: simple.hasOwnProperty(1); // true
