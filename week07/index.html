Functions: 1st Class Objects

with properties and methods that can be customized
As discussed previously, functions are 1st class objects in JS. More indepth discussion on: Function Methods Function Properties .call() : used to specify what 'this' points to in the function. Any functional arguments come after 'this'. It is very similar to: .apply() : also 'this' specifier, but takes an array of arguments to be passed in. call() Example:
function sayHi(){ return `Hi, I'm ${this.name}`; } let obj = {name: 'Crackers'}; sayHi.call(obj); // note the syntax-no () on f // also this does not work with arrow functions // evidently. ??
If method has arguments:
they come after 'this' arg in .call()
function sayHi(honorific){ return `Hi, ${honorific}. ${this.name}`; } let obj = {name: 'Crackers'}; sayHi.call(obj, 'Mr.'); "Goodbye, Mr. Crackers"
Now with .apply()
function sayBi(honorific) { return `Goodbye, ${honorific} ${this.name}`; } let obj = {name: 'Crackers'}; sayBi.apply(obj, ['Mr.']); // Even if just one element, must be in array // Goodbye, Mr. Crackers

IIFE

Immediately Invoked Function Expressions:
Used these in the Todo project for the Object Model to generate a Todo class with protected properties. An anonymous function enclosed within () and invoked as it's defined with a trailing '()' Variables kept within the scope of the function
// simple example: ( function (){ var loud = `boom`; console.log(loud); })() try { console.log(loud.length); } catch (refErr) { console.error('Is loud defined?', refErr); // loud is not defined }
But wait! Can't this just be done by placing a variable within a block? In ES6 Yes:
// ES6 Block: { let loud = `boom`; console.log(loud); } try { loud.length; } catch (refErr) { console.error('Is loud defined?', refErr); // loud is not defined }
